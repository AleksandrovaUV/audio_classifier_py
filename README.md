**Audio Processing Pipeline**

Этот проект, нацеленный на компексный анализ аудио и выполняющий:

- конвертацию аудио в нужный формат
- шумоподавление
- распознавание речи с таймкодами
- выделение тем разговора
- экспорт аудио и метаданных
- интерактивное CLI‑меню

---

## Структура репозитория

```
audio-pipeline/

```

---

# Описание модулей

## `main.py`: **Точка входа.**

Запускает CLI‑меню:

- спрашивает путь к входному файлу
- спрашивает каталог для вывода
- запускает пайплайн

## `ui/cli.py`: **Интерактивное меню на базе `rich`.**

Функции:

- ввод пути к файлу
- выбор имени выходного файла
- запуск пайплайна
- вывод итоговых путей

## `pipeline.py`

Он выполняет шаги строго по порядку:

1. Конвертация в WAV 16 kHz
2. Шумоподавление
3. Распознавание речи (ASR)
4. Тематический анализ
5. Экспорт аудио
6. Экспорт метаданных

Пайплайн использует конфигурацию из `config.yaml`.

## `converter.py: **Конвертация аудио в стандартный формат.**

Задачи:

- загрузить аудио любого поддерживаемого формата
- привести к моно
- привести к 16 kHz
- сохранить временный WAV

Использует `pydub` и `ffmpeg`.

## `cleanser.py`: **Шумоподавление.**

В текущей версии — простая нормализация громкости.  
Можно заменить на:

- RNNoise
- Demucs
- SpeexDSP
- Whisper pre‑processing

Модуль принимает WAV → возвращает очищенный WAV.

## `speech_to_text.py`: **Распознавание речи + таймкоды.**

Использует модель Whisper:

- загружает модель
- распознаёт речь
- возвращает список сегментов:

```json
[
  {"start": 0.0, "end": 4.2, "text": "Пример текста"},
  ...
]
```

## `topic_analyzer.py`: **Тематический анализ.**

Простая сегментация по блокам времени (каждые 60 секунд).  
Можно заменить на:

- sentence‑transformers embeddings
- HDBSCAN
- KMeans
- LDA
- LLM‑классификацию

Выход:

```json
[
  {
    "topic": "Topic block 0",
    "start": 0.0,
    "end": 60.0,
    "summary": "Краткое содержание блока…"
  }
]
```

## exporter.py`: **Экспорт результатов.**

Функции:

- экспорт финального аудио в выбранный формат (mp3, wav, flac…)
- экспорт метаданных в JSON

Метаданные включают:

- сегменты речи
- темы
- таймкоды

## `audio_utils.py`: **Низкоуровневая работа с аудио.**

Функции:

- загрузка аудио
- сохранение аудио
- приведение к моно
- приведение к 16 kHz

Используется конвертером и денойзером.

## `file_utils.py`: **Работа с путями и файлами.**

Функции:

- проверка существования входного файла
- создание директорий
- генерация путей для вывода

## `logger.py`: **Единый логгер.**

Используется всеми модулями для:

- информативных логов
- отладки
- отслеживания этапов пайплайна
